# AZ-IOT 菜单与交互逻辑重构方案 v3

> **核心原则：操作保持线性，每一步的完成自然引出下一步。**  
> **关键变更：采集器由客户端 MQTT 上线后自行注册，前端无需手动创建，只需查看状态和配置参数。**

---

## 一、系统数据流

```
BC-U101 采集端                 后端服务                    前端页面
─────────────                 ────────                   ────────
MQTT 连接上线 ──────────→ 自动创建/更新 collector 记录
                           生成 10 个 COM 口
                                 │
                                 │ GET /collectors
                                 ↓
                          采集器设置面板 ←── 显示状态、串口、采集周期
                                 │
                                 │ 用户新增设备
                                 ↓
                          选择采集器 → 加载串口 → 选择型号 → 创建
```

**采集器生命周期**：客户端注册 → 后端入库 → 前端只读展示 + 参数配置（周期、串口）。

```
┌─────────────────────────┐

```

**4 项菜单，无冗余。**

### 3.2 型号管理（`/device-model`）

**核心交互：新增弹窗内嵌点表导入。**

```
新增型号
  ── 基本信息 ──
  品牌 [▼ 北京昆仑海岸]
  协议 [▼ MODBUS_RTU    ]
  编码 [TH-802C_________]
  名称 [TH-802C 温湿度传感器__]
  描述 [____________________]

  ── 寄存器点表 ──
  ○ 稍后配置    ● 立即导入

  导入方式：[JSON 粘贴] [上传文件]

  ┌─ JSON 粘贴 ────────────────────────┐
  │ [{"sensorCode":"humidity",...},...] │
  └────────────────────────────────────┘

  预览：humidity:湿度(%RH)  dew_point:露点温度(℃)  ...  共6条

                              [取消] [创建]
```

- 选「立即导入」→ 粘贴/上传 → 预览 → 创建型号 + 自动 batchCreate 点表
- 选「稍后配置」→ 仅创建型号，点表留空，后续在点表管理页补齐
- 编辑模式：导入选项禁用，防止误覆盖

底部「下一步 → 注册设备」。

### 3.3 设备管理（`/device`）

页面分为两部分：

#### A. 采集器设置面板（可折叠，默认展开）

```
┌─ BC-U101 · MQTT 在线 ─────────────────────────── [收起] ─┐
│  采集周期: [600] 秒    [保存]                              │
│  客户端: neuron-collector-01 | IP: 192.168.1.100           │
│                                                           │
│  ── 串口列表 ──────────────────────────────────────────── │
│  串口  标签      类型   波特率  数据  停止  校验  启用  操作  │
│  COM1  ──      短信猫   ─      ─    ─     ─    ✓    ─   │
│  COM2  ──      IO板卡   ─      ─    ─     ─    ✓    ─   │
│  COM3  设备串口3 设备   9600    8    1    none  ✓   [编辑] │
│  COM4  ──       设备   9600    8    1    none  ✓   [编辑] │
│  ...                                                      │
└──────────────────────────────────────────────────────────┘
```

- 采集器信息只读，由客户端 MQTT 上线时自动注册
- 编辑采集周期 → 保存 → API 持久化
- 编辑串口 → 弹出 BusParamForm 结构化表单（波特率/数据位/停止位/校验位下拉框）→ 保存 → API 持久化 + MQTT 下发

#### B. 设备列表

```
[串口▼] [型号▼] [状态▼] [搜索...]       [新增设备]

前置条件：型号 ✓（3个）  ·  采集器 ✓（1台在线）

设备表格...
```

- 前置校验：无型号时「新增设备」置灰
- 底部「下一步 → 查看工作台」

#### C. 新增设备弹窗

```
编码      [EM-B1F-001              ]
名称      [一楼配电房温湿度传感器    ]
采集器    [▼ BC-U101 (N101)       ]   ← 下拉选择已注册采集器
串口      [▼ COM3                 ]   ← 联动加载该采集器的设备串口
型号      [▼ TH-802C              ]   ← 选型号自动填协议
协议      [MODBUS_RTU              ]   ← 禁用只读
从站地址   [1                      ]
采集间隔   [                       ]   ← 留空继承默认
                            [取消] [创建]
```

- 采集器列表从 GET /collectors 加载（均来自客户端自行注册）
- 选中采集器 → 加载其串口 → 选串口 → 选型号 → 填从站地址 → 创建

## 四、数据关联

```
dev_manufacturer                    dev_protocol
      │                                  │
      └──────┬───────────────────────────┘
             │
        dev_device_model  ←── dev_register_map (点表, 通过 model_id)
             │
             │ (用户在新增设备时应选择型号和采集器)
             │
        dev_device  ──serial_port_id──>  dev_serial_port
                                              │
                                         collector_id
                                              │
                                        dev_collector
                                         (客户端 MQTT 上线后自行注册, 无需手动创建)
```

**关键约束**：
- 一个设备绑定一个串口（`serial_port_id`）
- 一个串口挂在一个采集器下（`collector_id`）
- 同串口设备共享波特率，不同波特率设备必须挂不同 COM 口
- 采集器无需手动创建，客户端 MQTT 上线自动注册

## 五、用户操作流程

```
第1步 ①品牌管理
  → 录入厂商
  → 点「下一步」或侧栏进入型号管理

第2步 ②型号管理
  → 点「新增型号」
  → 选品牌 + 协议，填编码/名称
  → 选「立即导入」，粘贴 JSON 或上传文件
  → 预览点位 → 点「创建」
  → 点「下一步」或侧栏进入设备管理

第3步 ③设备管理
  → 展开「采集器设置」查看采集器状态和串口参数
  → 如有需要，编辑采集周期或串口波特率
  → 点「新增设备」
  → 选采集器 → 选串口 → 选型号 → 填从站地址 → 创建
  → 点「下一步」或侧栏进入工作台

第4步 ④工作台
  → 查看设备在线、数据采集
  → 日常监控
```
