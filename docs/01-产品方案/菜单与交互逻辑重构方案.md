# AZ-IOT 菜单与交互逻辑重构方案 v4

> **核心原则：操作保持线性，每一步自然引出下一步。**  
> **关键变更：采集器由客户端 MQTT 上线后自行注册，前端列表展示；设备新增按协议类型自适应（串行/TCP）。**

---

## 一、系统数据流

```
BC-U101 采集端                   后端服务                        前端页面
─────────────                   ────────                       ────────
MQTT CONNECT
  → ConnAck
    → publish status=online ──→ MqttSubscriberService         采集器列表 ← 状态实时
      定期 publish readings ──→ 写入 dev_device_reading
                                  更新 device.status=online
                                 │
                                 │ ConfigPushService
用户新增设备 ──────────────────→ 组装 config delta JSON
                                   ↓
            ← publish config ──→ MQTT neon/{clientId}/config/delta
采集器收到 → 注册设备 → 开始轮询 ─→ publish readings
```

**采集器生命周期**：客户端 CONNECT → ConnAck 发 online → 断连时 LWT 发 offline → 后端实时更新状态

### 3.2 型号管理（`/device-model`）

**核心交互：新增弹窗内嵌点表导入。**

```
新增型号
  ── 基本信息 ──
  品牌 [▼] 协议 [▼] 编码 [] 名称 [] 描述 []

  ── 寄存器点表 ──
  ○ 稍后配置    ● 立即导入

  导入方式：[JSON 粘贴] [上传文件]
  ┌─ textarea ──────────────┐
  │ [{sensorCode:...},...]  │    ← 支持串行/TCP 协议不同字段
  └─────────────────────────┘
  预览：temp:温度(℃)  flow:流量(m³/h)  ...  共N条

                              [取消] [创建]
```

**点表 JSON 按协议适配**：

串行协议（MODBUS_RTU）：
```json
[{"sensorCode":"humidity","sensorName":"湿度","register_address":0,"func_code":"0x04","data_type":"uint16","coefficient":0.1,"unit":"%RH","byte_order":"ABCD"}]
```

TCP 协议（MQTT/OPC UA/SNMP）使用 `extra_params`：
```json
[{"sensorCode":"temp","sensorName":"温度","data_type":"float","coefficient":1,"unit":"℃","extra_params":{"topic":"sensor/temp","json_path":"$.value"}}]
```

### 3.3 采集器（`/collector`）

**列表式**，点击名称展开设备子表：

```
┌──────────────────────────────────────────────┐
│ 采集器       采集器通过 MQTT 连接后自动注册    │
├──────────────────────────────────────────────┤
│ ▶ BC-U101  N101  aziot-collector-01  在线  编辑 下发配置 │
│   ├─────────────────────────────────────────┤
│   │ EM01  温湿度传感器  TH802C  COM5  在线  │
│   │              [管理全部设备 →]            │
│ └─────────────────────────────────────────┘
│ ▶ BC-U102  N102  collector-bak       离线  编辑 下发配置 │
└──────────────────────────────────────────────┘
```

- **编辑**：弹窗编辑名称/编码/mqttClientId/IP/周期/描述
- **下发配置**：遍历该采集器下所有设备，逐个 pushDelta
- **状态**：MConnAck → online，LWT → offline，MQTT 机制实时更新

---

### 3.4 设备管理（`/device`）

**按协议自适应**：

串行协议（MODBUS_RTU / DL_T645 / IEC101）：
```
编码 []  名称 []
采集器 [▼]  串口 [▼]    ← 串口必选
型号 [▼]    协议 [自动]  从站地址 []
                             [创建]
```

TCP 协议（MQTT / OPC_UA / BACnet / SNMP / HTTP_JSON...）：
```
编码 []  名称 []
采集器 [▼]                 ← 串口隐藏，无从站地址
型号 [▼]    协议 [自动]   连接方式：TCP/IP
                             [创建]
```

底部「下一步 → 工作台」。
