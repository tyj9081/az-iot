# 产品设计

> AZ-IOT Phase 0 产品方案 | 2026-06-25

---

## 项目边界

**做什么**: 品牌/型号/协议/点表管理 → 采集器/串口管理 → 设备绑定下发采集 → 数据展示

**不做什么**: 能耗分析、线损、BA楼控、报表、生产管理、光伏/空调——V2

---

## 核心用户故事

1. **运维工程师** 录入电表品牌（威胜）、型号（DTSD341）、协议（DL/T645），配置该型号的寄存器点表
2. **运维工程师** 注册 BC-U101 采集器，管理 8 个外接串口（COM3~COM10），将设备绑定到串口从站地址，一键下发
3. **运维工程师** 查看设备最新读数和历史趋势——电表显示数值卡片+趋势图，DI 设备显示状态指示灯

## 功能模块

### P1 基础设施
- Maven 多模块 + Cargo Workspace + Vue3 骨架
- 15 张 Flyway 建表脚本 (含分区表)
- JWT 双 Token 认证 (30min access + 7d refresh)
- 三角色 RBAC (admin/operator/viewer) + 审计日志

### P2 设备模板
- Protocol 字典管理 + Manufacturer CRUD
- DeviceModel CRUD (级联选择: 厂家→协议)
- RegisterMap CRUD (点表管理, 含 Excel 批量导入)

### P3 设备管理
- Collector CRUD (创建时自动生成 COM1-COM10 默认串口)
- SerialPort 配置管理
- Device CRUD (级联: 采集器→串口→型号→从站地址)
- MQTT 配置下发 (ConfigPushService)

### P4 采集端核心
- Protocol 驱动框架 (ModbusRTU/TCP/DL645)
- DeviceRegistry + 总线分组调度
- 双通道上报 (Channel A 实时, Channel B 降采样)
- 告警边缘判断引擎

### P5 数据展示
- Dashboard 概览页 (4 指标卡片 + 采集器状态 + 趋势图)
- 设备详情页 (通用渲染器: 数值卡片 / 状态灯)
- 历史趋势图

## 联动流程

```
创建设备:
  前端: 选厂家 → 选型号 → 选采集器 → 选串口 → 填从站地址
  服务端: INSERT dev_device → JOIN 三表组 JSON → MQTT delta
  采集端: 收到 ConfigDelta → DeviceRegistry.apply → 纳入调度

数据上报:
  通道A: 每次采集 → MQTT latest (QoS0) → WebSocket → 前端实时
  通道B: 降采样窗口 → avg/max/min → MQTT reading (QoS1) → MySQL → API

告警:
  采集端每次 poll → 本地阈值检查 → MQTT alarm → 服务端持久化 → 通知
```
