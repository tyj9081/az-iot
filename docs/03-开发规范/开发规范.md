## S01 — API 契约

### 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": { ... },
  "timestamp": 1782546982904
}
```

| code | 含义 |
|------|------|
| 0 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录 / 凭证无效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 资源冲突 (如编码重复) |
| 500 | 服务器内部错误 |

### 分页格式

```json
{
  "records": [...],
  "total": 100,
  "page": 1,
  "pageSize": 20
}
```

- page 从 1 开始 (非 0)
- JSON 字段统一 camelCase
- 日期格式 `yyyy-MM-dd HH:mm:ss`

### Controller 规范

- 路径: `@RequestMapping("/api/v1/{模块}")`
- **禁止** Controller 直接注入 Mapper 或拼接 SQL
- 入参必须用 DTO/VO 强类型，**禁止** `Map<String, Object>` 做入参
- 返回必须用 `ApiResponse<T>` 统一包装

## S02 — 数据库设计

### 表命名

- 业务表: `dev_` 前缀 (device, collector, protocol 等)
- 系统表: `sys_` 前缀 (user, role, permission 等)

### 字段规范

```sql
-- 主键: 必须 BIGINT AUTO_INCREMENT
id BIGINT AUTO_INCREMENT PRIMARY KEY

-- 审计字段: 每表必含
created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP

-- 逻辑删除: 需要软删除的表
is_deleted TINYINT NOT NULL DEFAULT 0

-- 状态字段: TINYINT + COMMENT 说明枚举值
status TINYINT NOT NULL DEFAULT 1 COMMENT '1=启用 0=禁用'
```

### 禁止事项

| ❌ 禁止 | ✅ 正确做法 |
|--------|------------|
| ENUM 类型 | VARCHAR + COMMENT 枚举值 |
| 关键字段 NULL | DEFAULT 明确值 |
| 外键无索引 | `INDEX idx_表_列 (column)` |
| Text/Varchar 无限制 | 明确长度上限 |

### Flyway 迁移

```
命名: V{序号}__{描述}.sql
示例: V1__init_sys_user.sql
      V11__add_mqtt_auth_to_dev_collector.sql
```

**铁律**: 
- **绝对禁止修改已执行的迁移文件**（Flyway 校验 checksum 会报错）
- 初始化数据的 INSERT 应使用 `ON DUPLICATE KEY UPDATE` 或先查后插，防止重复执行覆盖数据
- 版本号不可跳号不可重号

## S03 — XML Mapper (核心)

### 3.1 铁律: 所有 SQL 必须写入 XML

| ✅ 正确 | ❌ 禁止 |
|--------|--------|
| Mapper 接口 + XML 文件 | Service 层 `new LambdaQueryWrapper<>()` |
| XML `<select>` `<insert>` 标签 | `@Select` `@Insert` 注解写 SQL |
| `<foreach>` 批量操作 | for 循环逐条调 Mapper |
| `<if>` 动态条件 | Java 代码 `if + String.format` 拼 SQL |

### 3.2 目录结构

```
neuron-dao/
├── src/main/java/com/aziot/dao/mapper/
│   ├── system/SysUserMapper.java
│   └── device/DevDeviceMapper.java
└── src/main/resources/mapper/
    ├── system/SysUserMapper.xml     ← 一一对应，缺一不可
    └── device/DevDeviceMapper.xml
```

### 3.3 方法命名规范

| 操作 | 方法名 | XML 标签 |
|------|--------|---------|
| 按ID查 | `selectById` | BaseMapper 自带 |
| 按条件单条 | `selectByXxx` | `<select>` |
| 按条件列表 | `selectListByXxx` | `<select>` |
| 分页查询 | `selectPageByXxx` | `<select>` (传 Page 参数) |
| 插入 | `insert` | BaseMapper 自带 |
| 批量插入 | `insertBatch` | `<insert>` + `<foreach>` |
| 更新 | `updateById` / `updateXxx` | BaseMapper / `<update>` |
| 删除 | `deleteByXxx` | `<delete>` |
| 统计 | `countByXxx` | `<select>` |

### 3.4 分页查询返回类型

```java
// ✅ 正确: 返回 Page<T> (与调用方一致)
Page<DevDevice> selectPageByCondition(Page<DevDevice> page, ...);

// ❌ 错误: 返回 IPage<T> (导致 Service 层编译错误)
IPage<DevDevice> selectPageByCondition(Page<DevDevice> page, ...);
```

### 3.5 XML 编写关键规则

**OR 条件必须用括号包裹**:
```xml
<!-- ✅ 正确 -->
<if test="keyword != null and keyword != ''">
    AND (username LIKE CONCAT('%', #{keyword}, '%')
      OR nickname LIKE CONCAT('%', #{keyword}, '%'))
</if>

<!-- ❌ 错误: 没有括号，AND/OR 优先级错乱 -->
<if test="username != null">AND username LIKE ...</if>
<if test="nickname != null">AND nickname LIKE ...</if>
```

**SELECT 列名必须与 Entity 字段映射一致**:
```xml
<!-- ✅ 列名 = DB 实际列名，MyBatis-Plus 自动 camelCase 映射 -->
<sql id="Base_Column_List">
    id, username, password_hash, nickname, status, created_at, updated_at
</sql>
```

### 3.6 参数传递

```java
// 多个参数必须用 @Param 明确命名
List<SysUser> selectByCondition(
    @Param("keyword") String keyword,
    @Param("status") Integer status
);
```

### 3.7 方法归属

> SQL 操作哪张表，方法就定义在哪个 Mapper 上

```java
// ✅ 正确: 查 dev_device_model 表 → 方法在 DevDeviceModelMapper
public interface DevDeviceModelMapper {
    long countByManufacturerId(@Param("manufacturerId") Long manufacturerId);
}

// ❌ 错误: 查的是 dev_device_model 表，方法却定义在 DevManufacturerMapper
public interface DevManufacturerMapper {
    long countByManufacturerId(...);  // SELECT FROM dev_device_model
}
```

## S04 — Entity 实体规范

### 4.1 字段类型必须与 DB 列类型一致

| DB 类型 | Java 类型 | 说明 |
|---------|----------|------|
| TINYINT | `Integer` | 不可用 `String` |
| INT | `Integer` | |
| BIGINT | `Long` | |
| VARCHAR | `String` | |
| DECIMAL | `BigDecimal` | |
| DATETIME | `LocalDateTime` | |
| JSON | `String` (手动 parse) | |

### 4.2 @TableField 使用规则

```java
@Data
@TableName("sys_user")
public class SysUser {
    @TableId(type = IdType.AUTO)
    private Long id;

    private String username;
    
    // ✅ 让 MyBatis-Plus 自动映射: passwordHash → password_hash
    private String passwordHash;
    
    // ❌ 禁止: 覆盖 @TableField 导致与 DB 列名不一致
    // @TableField("password")   ← DB 里是 password_hash，映射失败！
    // private String passwordHash;
    
    private Integer status;  // ✅ TINYINT → Integer
    
    @TableLogic
    private Integer isDeleted;
    
    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createdAt;
    
    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updatedAt;
}
```

**铁律**: 
- `@TableField` 只在列名与自动映射不匹配时才用
- 使用前必须确认 DB 中实际列名与此一致
- 能用 `map-underscore-to-camel-case` 自动映射的不要加 `@TableField`

### 4.3 Lombok

- 统一使用 `@Data` (含 getter/setter/toString/equals/hashCode)
- 禁止手写 getter/setter

## S05 — Java Service 规范

### 5.1 注入方式

```java
// ✅ 构造器注入
@Service
@RequiredArgsConstructor
public class DevDeviceService extends ServiceImpl<DevDeviceMapper, DevDevice> {
    private final DevSerialPortMapper devSerialPortMapper;
}

// ❌ 禁止字段注入
@Autowired private DevSerialPortMapper mapper;
```

### 5.2 查询规范

```java
// ✅ 调用 Mapper 方法
public List<DevDevice> listByPortId(Long portId) {
    return baseMapper.selectByPortId(portId);
}

// ❌ 禁止: Service 层拼查询条件
public List<DevDevice> listByPortId(Long portId) {
    LambdaQueryWrapper<DevDevice> qw = new LambdaQueryWrapper<>();  // 禁止！
    qw.eq(DevDevice::getSerialPortId, portId);
    return list(qw);
}
```

### 5.3 事务

```java
@Transactional
public void create(DevDevice device) {
    // ...
}
```

- 写操作 (create/update/delete) 必须加 `@Transactional`
- 事务边界在 Service 层，不在 Controller

### 5.4 日志

```java
@Slf4j
public class XxxService {
    // ✅
    log.info("MQTT PUSH topic={}", topic);
    log.error("Config push failed for device {}", deviceId, e);
    
    // ❌ 禁止
    e.printStackTrace();
    System.out.println(...);
}
```

## S06 — Java Controller 规范

### 6.1 分层解耦

```
Controller  →  只做参数校验 + 调用 Service + 返回
Service     →  业务逻辑 + 事务
Mapper      →  数据库操作 (仅通过 XML)
```

```java
// ✅
@RestController
@RequestMapping("/api/v1/devices")
@RequiredArgsConstructor
public class DeviceController {
    private final DevDeviceService deviceService;

    @GetMapping
    public ApiResponse<Page<DevDeviceVO>> list(DevicePageReq req) {
        return ApiResponse.ok(deviceService.pageWithDetails(...));
    }
}

// ❌ 禁止: Controller 直接注入 Mapper
public class DeviceController {
    private final DevDeviceMapper deviceMapper;  // 禁止！
}
```

### 6.2 返回类型

```java
// ✅ Service 返回 VO，Controller 返回 VO
public ApiResponse<DevDeviceVO> getById(@PathVariable Long id) {
    return ApiResponse.ok(deviceService.getById(id));
}

// ❌ 禁止: Controller 暴露 Entity
public ApiResponse<DevDevice> getById(...) {
    return ApiResponse.ok(deviceMapper.selectById(id));  // 禁止！
}
```

### 6.3 异常处理

```java
// ✅ 抛业务异常，由全局 @RestControllerAdvice 统一处理
if (device == null) throw BusinessException.notFound("设备");

// ❌ 禁止: Controller 内 try-catch 吞异常
```

## S07 — MQTT 协议

- Topic: `neuron/{collector_mqtt_client_id}/{type}`
- type: `reading`, `latest`, `heartbeat`, `alarm`, `config/delta`, `config/full`, `config/ack`, `cmd`, `cmd/result`
- 荷载 JSON，**禁止包含 Java 包名或 Rust 模块路径**
- 通道A (latest): QoS 0
- 通道B (reading): QoS 1，批量 ≤500 条
- quality: 0=正常, 1=超时, 2=异常, 3=传感器故障

## S08 — Rust 代码

- `cargo clippy` 零警告，`cargo fmt` 统一格式
- **禁止 `unwrap()`** 在非测试代码 (用 `?` 或 `expect`)
- **禁止吞错误** `let _ = ...`
- **禁止 `println!`**，用 `tracing`
- Protocol trait: `async fn collect`，内部 `spawn_blocking` 包裹同步 IO
- 错误处理: `anyhow::Result` 顶层 + `thiserror` 结构化

## S10 — 安全规范

### 密码处理

```java
// ✅ 认证: BCrypt.matches(明文, DB中的hash)
if (!passwordEncoder.matches(rawPassword, user.getPasswordHash())) {
    throw new BusinessException(401, "用户名或密码错误");
}

// ❌ 禁止: 明文比较、MD5、SHA1、两次 hash 比较
if (!rawPassword.equals(user.getPasswordHash())) { ... }
```

### 凭证管理

- **绝对禁止** 硬编码密码/Token/Secret 在源码中
- 使用环境变量注入: `application-dev.yml` → `${DB_PASSWORD:}`
- `.env.local` / `application-dev.yml` 不提交 Git

### API 安全

- 登录失败不区分"用户不存在"和"密码错误"（防枚举攻击）
- JWT 中不存敏感信息 (密码、手机号等)
- Token 区分 access/refresh 类型，防止 access token 刷 refresh

## 附录A — 常见 Bug 清单

### A1. Entity 字段类型不匹配

```
症状: 登录始终 401，或字段值为 null
根因: DB 列 TINYINT → Java Entity 字段 String，类型转换失败
修复: TINYINT → Integer，比较用 != 1 而非 !"1".equals()
```

### A2. @TableField 列名不一致

```
症状: XML 能查到行，但 Entity 某些字段为 null
根因: @TableField("password") 但 DB 列名 password_hash
修复: 移除 @TableField，让 MyBatis-Plus 自动映射
```

### A3. Mapper 方法归属错误

```
症状: 编译报错 "找不到符号: 方法 xxx"
根因: 方法定义在 A Mapper 但 Service 调用在 B Mapper
修复: 方法归属 = SQL 查询的主表
```

### A4. IPage 返回类型编译错误

```
症状: IPage<T> 无法转换为 Page<T>
根因: Mapper 接口声明 IPage<T>，Service 声明 Page<T>
修复: Mapper 接口统一用 Page<T>
```

### A5. XML OR 条件缺括号

```
症状: 查询结果与预期不符 (OR 变成了 AND)
根因: <if> 独立块拼接 AND，缺少括号包裹 OR
修复: AND (col1 LIKE ... OR col2 LIKE ...)
```

### A6. Flyway 重复执行覆盖数据

```
症状: 后端重启后密码变了，登录不上
根因: 迁移文件被修改或 flyway_schema_history 被清，INSERT 重新执行
修复: init 数据用 INSERT ... ON DUPLICATE KEY UPDATE，且迁移文件不可修改
```
