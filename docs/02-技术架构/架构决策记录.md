## ADR-01 — 读数表按月分区

**状态**: Accepted

**Context**: dev_device_reading 按 30 秒采集周期，90 天可达 2.4 亿行。

**Decision**: 使用 MySQL RANGE PARTITION BY MONTH(read_at)，按月自动创建 + DROP 清理。

**Consequences**: 主键从 `(id)` 变为 `(id, read_at)` 联合主键。查询必须带 read_at 条件才能享受分区剪枝。

## ADR-02 — 数据保留两层

**状态**: Accepted

**Decision**:

| 层 | 存储 | 默认保留 | 配置 | 清理方式 |
|---|------|---------|------|---------|
| 采集端 | SQLite | 7 天覆盖 | config.toml | 定时 DELETE |
| 服务端 | MySQL 分区表 | 90 天 | sys_config API | DROP 分区 |
| 审计日志 | MySQL | 180 天 | sys_config API | 批量 DELETE |

**Consequences**: 两端独立管理，各自关注不同生命周期。

## ADR-03 — 双通道上报

**状态**: Accepted

**Decision**:

```
通道A (实时): latest (QoS0) → 服务端内存 → WebSocket → 前端 (<100ms)
通道B (历史): 降采样聚合 → reading (QoS1) → MySQL 分区表 → API
```

- 通道A 不进 DB，纯内存缓存 + WebSocket 广播
- 通道B 采集端本地 Aggregator 攒够窗口 → avg/max/min → 上报

**Consequences**: 实时感知和长期存储解耦。前端趋势图用 `avg` 画线，`max/min` 画误差带。秒级波形查看需直接访问采集端 SQLite。

## ADR-04 — 告警边缘判断

**状态**: Accepted

**Decision**: 采集端每次 poll 后本地阈值判断 (非服务端远程判断)。断网状态下 DO 继电器仍可联动声光报警。服务端只负责告警记录存储 + 通知分发。

**Consequences**: 告警延迟从 "采集→MQTT→服务端→判断" 缩短为 "采集→本地判断→MQTT alarm"。断网不盲。

## ADR-05 — 采集端降采样

**状态**: Accepted

| 采集周期 | 窗口 | MySQL 日写入 (47台×20点) |
|---------|------|------------------------|
| 1秒 | 60秒 | 27万 |
| 5秒 | 180秒 | 9万 |
| 10秒 | 300秒 | 5.4万 |
| ≥30秒 | 不聚合 | ≤28万 |

上报荷载: `{sensor_code, avg, max, min, sample_count, window_start, window_end}`

**Consequences**: MySQL 日写入量不随采集频率增长，始终 ≤28 万条。代价是前端看不到 1 秒级波形细节。

## ADR-06 — 模块化单体架构

**状态**: Accepted

**Decision**: Maven 技术层切模块(7 个), Java 包按业务域分包(device/collector/system)。不做微服务拆分。

**Consequences**: 部署简单，调试无网络开销。未来某域需要独立扩容时再拆分。
