## 模块架构

```
neuron-collector/
├── collector-model/           # 核心数据模型
│   ├── ProtocolType (18 种协议枚举)
│   ├── Device, DataPoint, ConfigDelta, LatestReading, AggregatedReading
│   └── BusType (serial/tcp), BusParam
├── collector-driver/          # 协议驱动层 (16 个真实驱动模块)
│   ├── ProtocolDriver trait (collect → HashMap<sensor_code, f64>)
│   ├── DriverFactory::create() (协议→驱动路由)
│   ├── modbus.rs         — Modbus RTU (tokio-modbus + 串口)
│   ├── modbus TCP        — Modbus TCP (tokio-modbus + socket)
│   ├── dlt645.rs         — DL/T645-1997/2007 (自实现645帧)
│   ├── iec104.rs         — IEC 60870-5-104 (APCI+ASDU)
│   ├── iec101.rs         — IEC 60870-5-101 (FT1.2帧)
│   ├── dnp3.rs           — DNP3 (CRC + Class 0)
│   ├── opcua.rs          — OPC UA (TCP Hello)
│   ├── mqtt_driver.rs    — MQTT订阅 (rumqttc)
│   ├── bacnet.rs         — BACnet/IP (BVLC + ReadProperty)
│   ├── snmp.rs           — SNMP v2c (BER GetRequest)
│   ├── http_poll.rs      — HTTP JSON (reqwest)
│   ├── canbus.rs         — CAN Bus (Linux SocketCAN)
│   ├── s7comm.rs         — Siemens S7 (ISO-on-TCP)
│   ├── profibus.rs       — PROFIBUS DP-V0
│   ├── ethernet_ip.rs    — EtherNet/IP CIP
│   ├── fins.rs           — OMRON FINS/TCP
│   └── mitsubishi.rs     — Mitsubishi MC
├── collector-scheduler/       # 调度引擎
│   ├── DeviceRegistry (HashMap<i64, Device>)
│   ├── bus_groups (按 bus_key 分组)
│   └── Collector::run() 主循环
├── collector-config-sync/     # MQTT 配置订阅
├── collector-reporter/        # 双通道上报
│   ├── RealtimePublisher  (通道A: QoS0, fire-and-forget)
│   ├── Aggregator (通道B: 降采样窗口, avg/max/min)
│   └── AlarmEngine (阈值告警 LOW/HIGH)
├── collector-storage/         # SQLite 本地缓存
├── collector-telemetry/       # 心跳 + metrics
└── collector-bin/             # 二进制入口
```

## 支持的协议 (18 种，全真实驱动)

### 串口协议 (6)
| 协议 | 传输 | 驱动模块 | 实现方式 |
|------|------|---------|---------|
| Modbus RTU | RS-232/485 | `modbus.rs` | tokio-modbus 库 |
| DL/T645-2007 | RS-485 | `dlt645.rs` | 自实现645帧 (BCB/BCD/CS) |
| DL/T645-1997 | RS-485 | `dlt645.rs` | 同上 (控制码差异) |
| IEC 60870-5-101 | RS-232 | `iec101.rs` | FT1.2 帧格式 |
| CAN Bus | CAN | `canbus.rs` | Linux SocketCAN (libc) |
| PROFIBUS DP | RS-485 | `profibus.rs` | DP-V0 Data_Exchange |

### TCP/IP 协议 (9)
| 协议 | 传输 | 驱动模块 | 实现方式 |
|------|------|---------|---------|
| Modbus TCP | Ethernet | `modbus.rs` | tokio-modbus 库 |
| IEC 60870-5-104 | TCP | `iec104.rs` | APCI+ASDU 帧 |
| DNP3 | TCP | `dnp3.rs` | CRC + Class 0 轮询 |
| OPC UA | TCP | `opcua.rs` | TCP Hello + 二进制解析 |
| BACnet/IP | UDP | `bacnet.rs` | BVLC + ReadProperty |

| 协议 | 传输 | 驱动模块 | 实现方式 |

| HTTP JSON | TCP | `http_poll.rs` | reqwest GET + JSON |

## 双通道数据上报

```
通道A (实时): 每次 poll → RealtimePublisher → MQTT QoS0
  Topic: neuron/{id}/latest
  荷载: {device_id, sensor_code, value, unit, read_at}
  不进 DB, 服务端纯内存 → WebSocket → 前端

通道B (历史): Aggregator 窗口攒够 → MQTT QoS1
  Topic: neuron/{id}/reading
  荷载: {device_id, sensor_code, avg, max, min, sample_count, window_start, window_end}
  入 MySQL 分区表 → API → 前端趋势图
```

## 告警引擎

支持 10 种工业物联告警类型，通过 `alarm_type` + `params(JSON)` 灵活扩展。

| 类型 | 说明 | 状态 |
|------|------|------|
| `limit_upper` | 上限告警 | ✅ 已实现 |
| `limit_lower` | 下限告警 | ✅ 已实现 |
| `limit_both` | 上下限告警 | ✅ 已实现 |
| `di_change` | DI 跳变 (0→1 / 1→0) | ✅ 已实现 |
| `timeout` | 通信超时 | ✅ 已实现 |
| `rate_rise` | 升速率 | 前端✅ Phase 2 |
| `rate_fall` | 降速率 | 前端✅ Phase 2 |
| `deviation` | 偏差告警 | 前端✅ Phase 2 |
| `deadband` | 死区告警 | 前端✅ Phase 2 |
| `custom` | 自定义表达式 | 扩展入口 |

- **配置下发**: 告警阈值随设备配置 MQTT delta 一并下发 → AlarmEngine.load_from_device() 热加载 → 下次 poll 生效
- **判断位置**: 采集端本地执行（非服务端），断网仍可 DO 联动声光
- **回滞机制**: limit_upper/lower/both 类型支持 hysteresis，避免边界抖动
- **防抖机制**: limit/rate/deviation 类型支持 delay_count，连续 N 次越限才触发

## 协议枚举与前后端对齐

采集端 `ProtocolType` 枚举与 Java 服务端 `ProtocolType.java` 通过 `code` 字段严格对应：

| Rust serde rename | Java enum code | DB code |
|-------------------|---------------|---------|
| `MODBUS_RTU` | `MODBUS_RTU` | `MODBUS_RTU` |
| `MODBUS_TCP` | `MODBUS_TCP` | `MODBUS_TCP` |
| `DL_T645_2007` | `DL_T645_2007` | `DL_T645_2007` |
| ... (全部 18 项逐字一致) | | |

新增协议流程：Rust enum → Java enum → Flyway INSERT → 驱动实现。
