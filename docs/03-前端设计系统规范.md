# AZ-IOT 前端 UI 设计系统规范

> **版本**: v1.0.0 | **制定日期**: 2026-06-27 | **强制执行**
>
> 本文档定义了 AZ-IOT 前端界面的**唯一合法**设计语言。所有新增页面、组件、样式修改必须严格遵循本规范。偏离本规范的代码应在 Code Review 阶段被拒绝。

---

## 一、设计哲学

### 1.1 产品定位

面向工业园区/楼宇的能源管理 IoT 平台。目标用户为运维工程师、设备管理员。

### 1.2 设计基调

| 维度 | 要求 |
|------|------|
| **专业感** | 深靛紫品牌色，高对比度数据展示，严谨间距体系 |
| **技术感** | 等宽数字字体、SVG 图表、精准的对齐与比例 |
| **可靠感** | 清晰的状态指示器、一致的交互反馈、无障碍访问 |
| **克制感** | 避免过度装饰，每个元素都必须有存在理由 |

### 1.3 核心原则

1. **设计令牌优先** — 所有颜色/间距/排版从 `tokens.css` 取值，**严禁硬编码**
2. **组件复用** — 提取公共模式为组件，禁止页面间复制粘贴样式
3. **无障碍内置** — WCAG AA 作为默认基线，不在上线前补救
4. **性能意识** — 动画仅用 `transform`/`opacity`，遵循 `prefers-reduced-motion`

## 二、设计令牌体系

> **源文件**: `src/styles/tokens.css`
> 所有 CSS 变量定义在此文件，通过 Element Plus 变量覆盖统一全局主题。

### 2.1 品牌色板

```
主色: #534AB7 (深靛紫) — 代表工业精度与数据洞察
```

| 色阶 | 色值 | 用途 |
|------|------|------|
| `--color-brand-50` | `#f3f0ff` | 背景、选中态浅底 |
| `--color-brand-100` | `#e8e2ff` | 悬停态浅底 |
| `--color-brand-200` | `#cec2ff` | 边框色 |
| `--color-brand-300` | `#ac97f5` | 图表辅助色 |
| `--color-brand-400` | `#8b6fe3` | hover 变体 |
| `--color-brand-500` | `#534AB7` | **主色**：按钮、链接、活跃态 |
| `--color-brand-600` | `#463ea0` | active 变体 |
| `--color-brand-700` | `#383289` | 深色背景文字 |
| `--color-brand-800` | `#2b2672` | 深色背景 |
| `--color-brand-900` | `#1e1b5b` | 最深色 |

### 2.2 语义色

| 语义 | 主色 | 50 色阶(背景) | 用途 |
|------|------|-------------|------|
| **成功** | `#10b981` (翡翠绿) | `#ecfdf5` | 在线状态、操作成功 |
| **警告** | `#f59e0b` (琥珀) | `#fffbeb` | 警告提示、禁用状态 |
| **危险** | `#ef4444` (胭脂红) | `#fef2f2` | 告警、错误、删除 |
| **信息** | `#3b82f6` (钢蓝) | `#eff6ff` | 中性信息、提示 |

### 2.3 中性灰阶

```
--color-gray-25:  #fafafb  (页面底色)
--color-gray-50:  #f5f5f8  (区块底色)
--color-gray-100: #eeeef3  (浅背景)
--color-gray-200: #e2e2ea  (边框)
--color-gray-300: #cacad4  (默认边框)
--color-gray-400: #a1a1ad  (占位文字)
--color-gray-500: #7b7b89  (次要文字)
--color-gray-600: #5a5a69  (正文)
--color-gray-700: #3f3f4d  (标题)
--color-gray-800: #282836  (深色标题)
--color-gray-900: #16161f  (最深)
```

**禁止使用纯黑 `#000` 或纯白 `#fff`** — 用灰阶替代。

### 2.4 排版规范

| 层级 | 字号 | 行高 | 字重 | 用途 |
|------|------|------|------|------|
| `text-xs` | 12px | — | 400/500 | 辅助标签、徽章文字 |
| `text-sm` | 14px | 1.5 | 400/500 | 次要文本、表格内容 |
| `text-base` | 16px | 1.5 | 400 | 正文、表单项 |
| `text-lg` | 18px | 1.5 | 500/600 | 小标题 |
| `text-xl` | 20px | 1.25 | 600 | 区块标题 |
| `text-2xl` | 24px | 1.25 | 600/700 | 页面标题 |
| `text-3xl` | 30px | 1.25 | 700 | 大标题 |
| `text-4xl` | 36px | 1 | 700 | 仪表盘指标数字 |

**字体家族**：
- 正文/UI: `Inter, -apple-system, BlinkMacSystemFont, system-ui, sans-serif`
- 等宽/数据: `JetBrains Mono, Cascadia Code, Consolas, monospace`

### 2.5 间距系统

基于 **8px 基础单位**，所有间距必须从以下取值：

```
--space-1:  4px    --space-2:  8px    --space-3:  12px
--space-4:  16px   --space-5:  20px   --space-6:  24px
--space-8:  32px   --space-10: 40px   --space-12: 48px
--space-16: 64px
```

**禁止使用 6px、10px、14px 等非 4 的倍数间距。**

### 2.6 圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `radius-sm` | 6px | 紧凑元素（标签、徽章） |
| `radius-md` | 8px | 按钮、输入框、小卡片 |
| `radius-lg` | 12px | 大卡片、内容区块 |
| `radius-xl` | 16px | 对话框、弹窗 |
| `radius-full` | 9999px | 药丸标签、状态指示器 |

### 2.7 阴影

| Token | 用途 |
|-------|------|
| `shadow-xs` | 常驻元素（header, sidebar） |
| `shadow-sm` | 卡片默认态 |
| `shadow-md` | 卡片 hover 态、下拉菜单 |
| `shadow-lg` | 弹出层 |
| `shadow-xl` | 对话框、模态框 |

### 2.8 动画

| Token | 缓动函数 | 时长 | 用途 |
|-------|---------|------|------|
| `ease-out-quart` | `cubic-bezier(0.25,1,0.5,1)` | 150ms | 按钮、hover |
| `ease-out-quint` | `cubic-bezier(0.22,1,0.36,1)` | 250ms | 弹窗、展开 |
| `ease-out-expo` | `cubic-bezier(0.16,1,0.3,1)` | 400ms | 页面过渡 |

**禁止**: `bounce`, `elastic`, `ease-in-out` 缓动函数。

## 三、布局规范

### 3.1 应用壳 (App Shell)

```
┌──────────────────────────────────────────────────┐
│ Header (56px) — Logo + 系统状态 + 用户操作         │
├──────────┬───────────────────────────────────────┤
│ Sidebar  │ Main Content                          │
│ (220px)  │ (padding: 24px)                       │
│          │                                       │
│ 导航菜单  │  <router-view />                      │
│ 带活跃态  │  页面过渡动画                           │
│          │                                       │
├──────────┴───────────────────────────────────────┤
└──────────────────────────────────────────────────┘
```

**Header**:
- 高度 56px
- 左侧: 品牌 Logo(32px SVG) + 应用名 + 副标题
- 右侧: 系统状态灯 + 退出按钮
- 背景 `surface-primary`, 底部 `1px solid border-light`, 阴影 `shadow-xs`

**Sidebar**:
- 宽度 220px
- 导航项: 14px, 500 字重, padding 10px 12px
- 活跃态: 左侧 3px 品牌色竖线 + 品牌色浅底
- 分组标签: 11px, 600 字重, 大写, 灰色
- 底部: 版本号标签

**Main**:
- padding: 24px
- 背景: `bg-page`

### 3.2 页面布局模式

所有管理类页面（品牌/型号/采集器/设备）必须使用以下结构：

```html
<div class="page-container">
  <!-- 工具栏行：筛选条件 + 操作按钮 -->
  <div class="page-toolbar">
    <div class="filter-group">
      <!-- 搜索框、下拉筛选等 -->
    </div>
    <el-button type="primary">新增XXX</el-button>
  </div>

  <!-- 表格 + 分页 -->
  <el-table .../>
  <el-pagination .../>
</div>
```

**禁止**使用自定义的 `.page-header` / `.header-left` / `.header-right` — 统一使用上述模式。

### 3.3 内容区块

详情页、仪表盘等多区块页面，使用 `.content-block`：

```html
<div class="content-block">
  <h3 class="section-heading">区块标题</h3>
  <!-- 区块内容 -->
</div>
```

**规则**:
- 区块间距: `margin-top: 16px` (通过相邻选择器自动添加)
- 区块标题: 14px/600 字重/大写/0.05em 字间距 + 左侧 3px 品牌色竖线
- 所有 `.content-block` 的 `h3` 必须使用 `.section-heading` 类

## 四、组件规范

### 4.1 状态指示器

四种标准状态，通过 `.status-indicator` + 状态类使用：

```html
<span class="status-indicator online">在线</span>
<span class="status-indicator offline">离线</span>
<span class="status-indicator alarm">告警</span>
<span class="status-indicator disabled">已禁用</span>
```

**视觉**: 8px 圆点 + 6px 间距 + 12px 文字。告警态带脉冲动画。

### 4.2 内联状态徽章

用于表格列内的状态展示（替代 el-tag）：

```html
<span class="inline-status online">在线</span>
```

使用 11px 文字、药丸形状、对应语义色背景。

### 4.3 等级徽章

告警等级专用：

```html
<span class="level-badge critical">严重</span>
<span class="level-badge warning">警告</span>
<span class="level-badge info">提示</span>
```

### 4.4 指令类型标签

```html
<span class="instr-type READ">读取</span>
<span class="instr-type WRITE">写入</span>
<span class="instr-type CONTROL">控制</span>
<span class="instr-type CONFIG">配置</span>
```

### 4.5 读数卡片（设备详情页）

三种渲染模式，根据 `data_type` 自动切换：

| 数据类型 | 渲染组件 | 视觉特征 |
|---------|---------|---------|
| `float32/64`, `int16/32`, `uint16/32` | 数值卡片 | 24px 等宽粗体 + 单位 + 质量指示 |
| `bool` | 状态灯卡片 | 32px 圆灯(发光) + ON/OFF + 可选开关 |
| 其他 | 文本卡片 | 16px 普通文字 |

卡片容器: `surface-secondary` 背景、`radius-md` 圆角、hover 时边框高亮 + 上浮 1px。

### 4.6 指标卡片（仪表盘）

4 种语义色变体，通过 CSS 类区分：

```html
<div class="metric-card metric-card--total">   <!-- 品牌紫 -->
<div class="metric-card metric-card--online">  <!-- 翡翠绿 -->
<div class="metric-card metric-card--readings"><!-- 钢蓝 -->
<div class="metric-card metric-card--alarm">   <!-- 胭脂红 -->
```

每种变体:
- 独立的 SVG 图标色
- 顶部装饰线（hover 时展开）
- `.metric-card--alarm` 额外有：边框高亮 + 右上角脉冲动画

## 五、Element Plus 覆盖规则

所有 Element Plus 组件视觉通过 `src/styles/tokens.css` 中的 CSS 变量统一覆盖。**禁止**在业务页面中单独覆盖 Element Plus 组件样式（统一到 tokens.css 或 element-override.css 中处理）。

### 5.1 已覆盖组件清单

| 组件 | 覆盖项 |
|------|--------|
| `el-button` | 品牌色、圆角、hover 微位移 + 阴影 |
| `el-input` | 聚焦时品牌色发光边框 |
| `el-table` | 品牌色 hover 行、表头底色 |
| `el-tag` / `el-select` / `el-dialog` / `el-menu` / `el-pagination` | 品牌色全局统一 |

### 5.2 新增覆盖流程

当发现 Element Plus 组件与设计规范不一致时：
1. 在 `element-override.css` 中添加全局覆盖规则
2. 使用 CSS 变量（非硬编码色值）
3. 在 Code Review 中注明覆盖原因

## 七、新增页面 Checklist

开发新页面时，必须逐项确认：

- [ ] 使用 `.page-container` 作为外层容器
- [ ] 工具栏使用 `.page-toolbar` + `.filter-group`
- [ ] 所有颜色值从 CSS 变量引用（非硬编码 hex）
- [ ] 所有间距使用 `--space-*` 变量（非任意 px 值）
- [ ] 表格使用 Element Plus 组件（不自行实现）
- [ ] 状态展示使用规范组件（`.status-indicator` 系列）
- [ ] 数字使用 `font-variant-numeric: tabular-nums` + 等宽字体
- [ ] 色彩对比度 ≥ 4.5:1（WCAG AA 级）
- [ ] 动画仅使用 `transform`/`opacity`，遵守 `prefers-reduced-motion`
- [ ] 页面有进入/离开过渡动画
- [ ] `<style scoped>` 不包含全局规则
- [ ] 无硬编码的 `#fff`、`#000`、`#eee`

## 八、禁止行为

| 违规 | 说明 |
|------|------|
| **硬编码色值** | 禁止在业务样式中写 `#534AB7`、`#f00` 等，必须用 `var(--color-brand-500)` |
| **自定义间距** | 禁止 `margin: 15px`、`padding: 7px`，必须用 `--space-*` |
| **直接覆盖 EP 组件** | 禁止在业务 `<style scoped>` 中 `:deep()` 覆盖 el-button/el-table 等 |
| **纯黑/纯白** | 禁止 `#000`、`#fff`，使用灰阶 `--color-gray-*` |
| **bounce/elastic 动画** | 禁止弹跳缓动，只用 quart/quint/expo |
| **样式复制粘贴** | 相同模式提炼为组件或 CSS 类 |
| **新字体引入** | Inter + JetBrains Mono 已足够，未经审批不得新增 |
